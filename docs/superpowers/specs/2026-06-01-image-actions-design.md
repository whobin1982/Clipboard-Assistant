# 剪贴板助手 0.4.0 图片右键操作设计

## 目标

0.4.0 聚焦 GitHub issue #8：给图片历史项增加右键操作，让图片记录不仅能点击粘贴，也能直接复制图片、复制图片文字、导出图片。这个版本保持范围克制，不实现 OCR 识别本体；OCR 会在后续 #7 中单独完成。

## 用户体验

用户在剪贴板历史窗口中右键点击图片记录时，看到图片专用菜单：

- 复制图片
- 复制图片文字
- 导出图片

“复制图片”把图片记录写回系统剪贴板，和现有复制按钮使用同一套图片恢复逻辑。

“复制图片文字”在 0.4.0 中先提供明确反馈。如果图片记录还没有 OCR 文本，显示中文提示“这张图片还没有可复制的识别文字”。后续 #7 增加 OCR 字段后，这个菜单项再复制 OCR 文字。

“导出图片”打开 macOS 保存面板。默认文件名使用“剪贴板图片”，默认扩展名优先使用 PNG。用户选择位置后，应用把图片记录中的第一张可解码图片导出为 PNG。取消保存不报错。

## 范围

包含：

- 图片行右键菜单。
- 复制图片动作。
- 无 OCR 文本时的复制图片文字提示。
- 图片导出保存面板和 PNG 导出。
- README、帮助文档或发布说明中的功能说明。
- v0.4.0 版本号、安装包和 GitHub Release。

不包含：

- OCR 识别、OCR 文本存储和 OCR 搜索。
- 图片批量导出。
- 图片格式选择器。
- 自动更新。

## 架构调整

### 行视图

`ClipboardRowView` 保持负责单条记录的展示和交互。为图片记录增加上下文菜单，菜单回调由外层注入，避免行视图直接知道系统剪贴板、保存面板和错误展示细节。

新增回调建议：

- `onCopyImageText`
- `onExportImage`

文本记录不显示图片菜单。

### 历史窗口

`ClipboardPopupView` 继续作为列表容器，把图片菜单动作从行视图转交给 `SearchWindowPresenter` / `AppEnvironment` 注入的业务闭包。窗口本身不处理文件导出细节。

### 图片导出服务

图片导出应复用现有 `ClipboardImageArchive`。新增轻量服务，负责：

1. 从图片记录读取 `imagePath`。
2. 加载图片归档或兼容旧版单图片文件。
3. 取第一张可解码图片。
4. 将图片写为 PNG 到用户选择的目标位置。

保存面板是 UI 边界，适合放在 AppKit 层或服务层注入，测试中用假的导出目标替代真实面板。

### 错误处理

所有失败都通过现有 `lastErrorMessage` 显示中文提示。

- 图片路径缺失：提示“这条图片记录缺少图片文件路径。”
- 图片不可读取：沿用现有不可读图片提示。
- 没有 OCR 文本：提示“这张图片还没有可复制的识别文字。”
- 导出失败：提示“导出图片失败，请换一个位置后再试。”
- 用户取消导出：不显示错误。

## 测试

新增或更新测试覆盖：

- 图片记录右键菜单包含复制图片、复制图片文字、导出图片。
- 文本记录不显示图片专用菜单。
- 复制图片文字在没有 OCR 文本时设置中文错误提示。
- 导出图片可以把图片记录写成 PNG 文件。
- 用户取消导出不显示错误。
- 图片导出失败时显示中文错误提示。
